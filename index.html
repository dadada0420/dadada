📊 HTML vs JSX 차이점 완벽 정리

웹 개발을 하다 보면 가장 자주 마주치는 두 가지 개념인 HTML과 JSX의 핵심 차이점을 깔끔하게 정리한 문서입니다.

1. 한눈에 보는 비교 표

특징 HTML (.html) JSX (.jsx / .js)
정의 웹 표준 마크업 언어 자바스크립트 확장 문법 (주로 React에서 사용)
브라우저 실행 브라우저가 직접 읽고 화면에 표시 가능 브라우저가 직접 못 읽음 (JS 변환 필요)
자바스크립트 결합 <script> 태그를 통해 외부에서 조작 UI 코드 내에 JS 로직을 직접 삽입 가능 ({ } 사용)
클래스 지정 속성 class="container" className="container"
태그 닫기 규칙 <img>, <br> 등 생략 가능 모든 태그는 반드시 닫아야 함 (<img />)

2. 핵심 차이점 3가지

① 브라우저의 직접 해석 여부

HTML은 크롬, 사파리, 엣지 등 모든 웹 브라우저가 태생적으로 직접 이해하고 화면에 그려낼 수 있는 웹의 기본 언어입니다.

반면 JSX는 브라우저가 그대로 읽으면 문법 에러가 발생합니다. 따라서 빌드 과정에서 Babel(바벨) 같은 컴파일러를 통해 순수한 자바스크립트 객체 코드로 변환되어 실행됩니다.

② 자바스크립트(JS)와의 결합도

HTML은 화면을 그리는 역할만 담당하므로, 데이터를 동적으로 바꾸려면 별도의 자바스크립트 파일에서 DOM을 하나하나 찾아서 조작해야 합니다.

JSX는 UI와 로직이 한 몸으로 움직입니다. JSX 내부에서 중괄호 { }만 열면 변수, 함수, 삼항 연산자 등의 자바스크립트 코드를 자유롭게 섞어 쓸 수 있어 동적인 화면 표현이 매우 강력합니다.

③ 엄격한 문법 규칙

JSX는 겉보기엔 HTML 같지만 본질은 '자바스크립트'이기 때문에 몇 가지 독자적인 규칙이 있습니다.

  • className 사용: JS에는 이미 클래스를 정의하는 class 키워드가 있으므로, 중복을 피하기 위해 HTML의 class 대신 className을 사용합니다.
  • 카멜 케이스(CamelCase): HTML의 onclick, onchange 속성은 JSX에서 onClick, onChange처럼 대소문자를 구분하여 작성해야 합니다.
  • Self-closing 태그: HTML에서는 <input>이나 <br>을 닫지 않아도 브라우저가 유연하게 넘어가지만, JSX에서는 무조건 <input />, <br />와 같이 닫아주어야 에러가 나지 않습니다.
💡 요약

HTML은 브라우저가 바로 읽는 웹의 기본 문서이고, JSX는 현대적인 웹 프레임워크(React 등)에서 화면을 더 스마트하고 다이내믹하게 만들기 위해 자바스크립트에 HTML 옷을 입혀놓은 것입니다.
